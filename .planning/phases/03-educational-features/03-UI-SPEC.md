# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Educational Features.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — raw React + inline styles (matches Phase 1/2 pattern) |
| Icon library | none — text labels only, unicode symbols where needed (e.g. "+" for add) |
| Font | Inter, Avenir, Helvetica, Arial, sans-serif (inherited from App.css :root) |

No shadcn. All components use inline `style` objects (not Tailwind utility classes) consistent with BottomToolbar.tsx, HandStatusIndicator.tsx, and PrePermissionScreen.tsx from prior phases.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicator gaps, chip inner padding top/bottom |
| sm | 8px | Chip horizontal padding, label bubble padding, toolbar button gaps |
| lg | 16px | Toolbar height padding, drawer padding, chip row padding |
| xl | 24px | Drawer header padding, section vertical separation |
| 2xl | 48px | Drawer width is NOT in this scale — see Surfaces section |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Toolbar minimum height: 44px (touch target floor, carried from Phase 2)
- Toolbar button height: 36px (carried from Phase 2)
- md: 12px — used for drawer item padding and toolbar button horizontal padding — midpoint between 8px and 16px required to match existing BottomToolbar.tsx button padding pattern carried from Phase 2
- Drawer width: 280px on desktop / 100vw on viewport < 400px
- Label bubble max-width: 240px

## Color

All surfaces live at z:10 over the live webcam feed. Semi-transparency is mandatory so the AR canvas remains visible underneath.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | rgba(17, 24, 39, 0.7) | Toolbar background, drawer background, label bubble background — matches existing toolbar (background: 'rgba(17, 24, 39, 0.7)') |
| Secondary (30%) | rgba(255, 255, 255, 0.08) | Drawer item hover state, active model highlight fill, chip toggle background (inactive) |
| Accent (10%) | #2563EB | Active/selected state only — see reserved list below |
| Destructive | #7F1D1D bg / #FCA5A5 text | Error toast only — matches Phase 2 error toast exactly |

Accent (#2563EB) reserved for:
- Active model card highlight border in drawer
- Chip toggle background when layer is ON (visible)
- Inspect mode button background when active
- Explode button background when active

Accent is NOT used for: hover states, disabled states, decorative borders, or any always-visible UI chrome.

Additional semantic colors (status indicators only, not accent):
- Hand detected dot: #22C55E (green-500) — carried from Phase 2 HandStatusIndicator
- Hand not detected dot: #6B7280 (gray-500) — carried from Phase 2 HandStatusIndicator
- Loading spinner border-top: #2563EB — carried from Phase 2 spinner

Text colors:
- Primary text on dark overlays: #ffffff
- Secondary/muted text: rgba(255, 255, 255, 0.6)
- Active chip text: #ffffff
- Inactive chip text: rgba(255, 255, 255, 0.7)

Border colors:
- Toolbar dividers: rgba(255, 255, 255, 0.2) — carried from Phase 2
- Inactive button border: rgba(255, 255, 255, 0.3) — carried from Phase 2
- Active model border in drawer: #2563EB, 2px solid
- Label bubble border: rgba(255, 255, 255, 0.15)
- Chip border inactive: rgba(255, 255, 255, 0.2)

## UI Surfaces

### Surface 1: Model Gallery Drawer

**Trigger:** "Models" button in bottom toolbar (replaces existing "Load Model" button)
**Direction:** slides in from the LEFT edge
**Width:** 280px (desktop); full width on viewport < 400px
**Background:** rgba(17, 24, 39, 0.85) with `backdropFilter: blur(8px)`
**Z-index:** 10 (same layer as toolbar)
**Backdrop:** fixed overlay behind drawer, rgba(0, 0, 0, 0.4), covers full viewport — canvas keeps running underneath

**Header:**
- Text: "Select Model" — 16px / 600 / #ffffff
- Padding: 16px all sides
- Bottom border: rgba(255, 255, 255, 0.1), 1px solid

**Model list items:**
- Layout: row, gap 12px, align items center
- Padding: 12px 16px (12px is the spacing exception — see Spacing Scale Exceptions)
- Thumbnail: 56px x 56px, borderRadius 6px, objectFit cover; for user-loaded files show unicode fallback "[ ]" (box character) at 28px in a 56x56 dark background
- Model name: 14px / 400 / #ffffff
- Active item: left border 3px solid #2563EB, background rgba(37, 99, 235, 0.15)
- Hover state: background rgba(255, 255, 255, 0.08)

**"+ Load file" entry (bottom of list):**
- Separator line: rgba(255, 255, 255, 0.1), 1px solid, 8px margin top/bottom
- Text: "+ Load file" — 14px / 400 / rgba(255, 255, 255, 0.7)
- No thumbnail — text only, left-aligned at x:16px

**Dismiss:** clicking the backdrop overlay closes the drawer; clicking a model entry loads it and closes immediately

### Surface 2: Body Part Label Bubble

**Anchor:** drei `<Html>` component at selected mesh's world position — projects with the model in 3D space
**Offset:** 16px above the mesh center point (drei `<Html center>` with a `style.transform` translateY -50%)
**Max width:** 240px
**Background:** rgba(17, 24, 39, 0.85) with `backdropFilter: blur(6px)`
**Border:** 1px solid rgba(255, 255, 255, 0.15)
**Border radius:** 8px
**Padding:** 12px (spacing exception — see Spacing Scale Exceptions)

**Internal layout:**
- Anatomy name: 16px / 600 / #ffffff — top line
- Description text: 12px / 400 / rgba(255, 255, 255, 0.8) — below name, line-height 1.5
- Pin indicator: a 6px wide vertical line on left edge of bubble, #2563EB — visually anchors the label

**Pointer arrow:** 8px wide downward-pointing triangle in rgba(17, 24, 39, 0.85) at bubble bottom-center, connecting to the mesh anchor point

**Dismiss:** another point+hold on a different mesh replaces the label; point+hold on empty space dismisses

### Surface 3: Layer Toggle Chip Row

**Trigger:** "Layers" button in bottom toolbar
**Position:** expands ABOVE the bottom toolbar as an inline row
**Container background:** rgba(17, 24, 39, 0.7) with `backdropFilter: blur(4px)`
**Container padding:** 8px 16px
**Container min-height:** 44px
**Gap between chips:** 8px
**Overflow:** horizontal scroll if chips exceed viewport width

**Chip anatomy:**
- Padding: 4px 8px
- Border radius: 16px (pill shape)
- Font: 12px / 400
- Active (visible): background #2563EB, text #ffffff, no border
- Inactive (hidden): background rgba(255, 255, 255, 0.08), text rgba(255, 255, 255, 0.6), border 1px solid rgba(255, 255, 255, 0.2)
- Height: 28px

**Chip labels:** use prettified group name — capitalize first letter, replace hyphens with spaces
(e.g., "left-arm" → "Left Arm", "ribcage" → "Ribcage")

**If no named groups exist on the loaded model:** "Layers" button is disabled — opacity 0.4, cursor not-allowed, no chip row

## Interaction Contracts

### Pointing gesture → label (MDL-03)

| State | Visual |
|-------|--------|
| No gesture | No label bubble visible |
| Pointing at mesh, hold < 1s | No visual change (dwell timer silent) |
| Pointing at mesh, hold >= 1s | Label bubble appears anchored to mesh |
| Pointing at new mesh | Previous label fades out immediately, new label appears |
| Pointing at empty space, hold >= 1s | Current label dismisses, nothing new shown |

No loading indicator during the 1-second dwell — the hold period is short enough that a spinner would feel laggy.

### Explode toggle (MDL-05)

| State | Visual |
|-------|--------|
| Explode OFF | Model parts at natural positions; "Explode" button ghost style |
| Explode ON (animating) | Parts lerp to exploded positions over ~0.5s; button switches to accent fill immediately on click |
| Explode ON (settled) | Parts at 1.5× bounding radius offset positions; button stays accent fill |
| Explode OFF (animating) | Parts lerp back; button reverts to ghost immediately on click |

### Inspect mode (D-20 to D-24)

| State | Visual |
|-------|--------|
| Inspect OFF | "Inspect" button ghost style |
| Inspect ON | "Inspect" button accent fill (#2563EB) |
| Inspect ON + spread fingers detected | Explode activates (same as tapping Explode button) |
| Inspect ON + fist detected | Explode deactivates |
| Inspect ON + pinch drag | Normal rotation, unchanged |

### Drawer open/close

| Event | Behaviour |
|-------|-----------|
| Tap "Models" button | Drawer slides in from left, backdrop fades in over 150ms |
| Tap backdrop | Drawer slides out to left, backdrop fades out over 150ms |
| Tap model entry | Model loads immediately, drawer closes |
| Tap "+ Load file" | File picker opens, drawer stays open until file selected or cancelled |

## Accessibility Notes

- All toolbar buttons must have a visible focus ring when navigated via keyboard (outline: 2px solid #2563EB, outline-offset: 2px)
- Touch targets: all interactive elements minimum 44×44px hit area (button height 36px padded to 44px touch area via surrounding container)
- Drawer items: minimum 48px tall for touch friendliness
- Layer chips: minimum 28px tall — acceptable for supplementary controls in a desktop-primary app
- Label bubbles: `pointer-events: none` on the drei Html wrapper to avoid blocking 3D interaction; dismiss via gesture only
