  </read_first>
  <action>
    Create a platform detection module that enables conditional imports:
    
    **src/config/environment.ts:**
    Export a constant that detects Tauri runtime: `export const IS_TAURI = typeof window !== 'undefined' && '__TAURI_CORE__' in window;`
    Also export: `export const IS_WEB = !IS_TAURI;`
    
    **src/utils/platform.ts:**
    Create helper function: `export function onlyTauri<T>(fn: () => T): T | null { return IS_TAURI ? fn() : null; }`
    This allows code like: `const path = onlyTauri(() => import('@tauri-apps/api').then(m => m.path))`
    
    **vite.config.ts — Add environment handling:**
    Ensure the config properly defines `__TAURI_CORE__` during Tauri builds. The template should do this automatically, but verify the Tauri Vite plugin is enabled.
    
    **Usage pattern (document for downstream tasks):**
    Instead of: `import { someApi } from '@tauri-apps/api'` at top level
    Do this: `if (__TAURI_CORE__) { const { someApi } = await import('@tauri-apps/api'); }`
    
    This allows web builds to tree-shake Tauri imports completely.
    
    Verification: Build for web and Tauri separately. In web build, @tauri-apps/api should not appear in bundle. In Tauri build, __TAURI_CORE__ should be true.
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vite dev server → Browser | HMR and dev assets served locally; no untrusted input |
| Tauri webview ← OS | Tauri webview inherits OS security; CSP restricts asset loading |
| App ← User input | User interactions will drive gesture/input in later phases; current phase has no user-submitted data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-PLAN-01 | Tampering | tauri.conf.json security config | mitigate | Ensure CSP restricts script-src to 'self' only; no eval() or unsafe-inline. Define content-security-policy in tauri.conf.json with default-src 'self', script-src 'self', style-src 'self' 'unsafe-inline' (Tailwind needs unsafe-inline), connect-src 'self' for HMR in dev. |
| T-01-PLAN-02 | Information Disclosure | __TAURI_CORE__ detection | accept | __TAURI_CORE__ is a compile-time constant, not user-exposed. No PII or secrets in environment detection. |
| T-01-PLAN-03 | Denial of Service | pnpm-lock.yaml | mitigate | Commit pnpm-lock.yaml to version control. Verify package integrity via pnpm's built-in audit. Run `pnpm audit` before each phase. |

</threat_model>
