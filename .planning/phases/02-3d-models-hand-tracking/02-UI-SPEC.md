| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline dot padding |
| sm | 8px | Toolbar item gap, button inner spacing (x-axis) |

Accent (`#2563EB` blue-600) is reserved for:
1. "Load Model" primary CTA button
2. Active pinch landmark dots (thumb tip + index fingertip when pinch threshold met)
3. Hover/focus state on toolbar buttons

Semantic colors:
- Hand detected indicator dot: `#22C55E` (green-500) — hand present
- Hand not detected indicator dot: `#6B7280` (gray-500) — no hand

## Component Inventory

### Bottom Toolbar

**Location:** Fixed bottom of viewport, full viewport width, z-index 10
**Layout:** Horizontal flex row, items centered vertically, 16px horizontal padding, 44px height minimum
**Background:** `rgba(17, 24, 39, 0.7)` with `backdrop-filter: blur(4px)`
**Items (left to right):**
1. "Load Model" button — blue-600 filled, 12px label at weight 400, 8px x-padding, 44px touch target
2. Separator — 1px vertical divider, gray-700
3. Landmark toggle button — icon + label "Landmarks ON/OFF" at weight 400, toggles `landmarksVisible` state

### Hand Detection Status Indicator

**Location:** Top-right corner, fixed, z-index 10, 8px from edges
**Layout:** Flex row, 4px gap — colored dot (8px circle) + text label
**States:**
- Loading: gray dot + "Loading hand tracking..." at 12px/400
- No hand: gray-500 dot + "No hand detected" at 12px/400
- Hand detected: green-500 dot + "Hand detected" at 12px/400

### Landmark Canvas

**Location:** Between webcam video (z:0) and R3F canvas (z:1). Use `z-index: 0.5` via explicit `style` prop — class `.z-1` does not support fractional. Use `style={{ zIndex: 0.5 }}` inline.
**Dimensions:** Match `window.innerWidth` x `window.innerHeight`. Update on resize.
**Rendering:** `transform: scaleX(-1)` to mirror horizontally (matches mirrored webcam feed per D-27)
**Landmark dot:** 4px radius filled circle
- Default color: `rgba(255, 255, 255, 0.7)`
- Pinch-active (thumb tip landmark 4 + index fingertip landmark 8): `#2563EB` (blue-600)
- No connecting lines (per D-09)

### Loading Spinner Overlay

**Location:** Centered over R3F canvas, z-index 10
**Background:** `rgba(17, 24, 39, 0.8)` — semi-transparent dark
**Content:** Animated spinner (CSS border-radius spin, 32px, blue-600 border) + "Loading model..." text at 20px/700 white
**Visibility:** Shown during GLB parse/load. Hidden when model is ready.

### Error Toast

**Location:** Bottom-right, fixed, 16px from edges, z-index 20
**Background:** `#7F1D1D` (red-900)
**Text:** `#FCA5A5` (red-300), 14px/400, max-width 320px
**Duration:** Auto-dismiss after 5 seconds
**Copy:** See Copywriting Contract

## Interaction Contract

### Mouse Controls (OrbitControls)
- Left-drag: rotate model (trackball style)
- Scroll wheel: zoom in/out
- Right-drag: pan
- Active when: no pinch gesture detected, or hand tracking not yet loaded

### Gesture Controls (MediaPipe)
- Single-hand pinch + drag: rotate (trackball style, D-17)
- Dead zone: 10px movement before rotation triggers (D-19)
- Two-hand pinch: scale — distance between pinch points maps to model scale (D-18)
- Two-hand same-direction movement: pan (D-20)
- Auto-switch: OrbitControls disabled when pinch detected; re-enabled after 0.5s of no pinch (D-23, D-25)
- Momentum/inertia after pinch release: smooth deceleration over 300ms (Claude's discretion — chosen for anatomy inspection; instant stop interrupts spatial understanding of the model)

### Model Auto-Fit (D-06)
- On GLB load: calculate bounding box, center model at origin, scale to fit within a 2-unit radius sphere
- Auto-rotation (from Phase 1) stops permanently when a real GLB model is loaded or user first interacts

### File Picker (D-02)
- "Load Model" button triggers `<input type="file" accept=".glb,.gltf">` programmatically
- Replaces current model on selection (D-03)
- Shows spinner overlay during load (D-07)
- On failure: shows error toast, keeps previous model (D-08)
