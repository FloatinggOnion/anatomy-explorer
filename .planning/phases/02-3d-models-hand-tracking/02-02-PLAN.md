```tsx
import { Box3, Vector3 } from 'three';
// In useEffect after GLTF load:
const box = new Box3().setFromObject(groupRef.current);
const center = box.getCenter(new Vector3());
groupRef.current.position.sub(center);
const maxDim = Math.max(...box.getSize(new Vector3()).toArray());
if (maxDim > 0) groupRef.current.scale.setScalar(2 / maxDim);
// Reset OrbitControls target (Pitfall B):
if (controlsRef?.current) { controlsRef.current.target.set(0,0,0); controlsRef.current.update(); }
```

File picker + object URL pattern (from 02-PATTERNS.md):
```tsx
// Track prev URL in a ref (not Zustand) for revocation:
const prevObjectUrlRef = useRef<string | null>(null);
const handleFileChange = (e) => {
  const file = e.target.files?.[0];
  if (!file) return;
  const newUrl = URL.createObjectURL(file);
  if (prevObjectUrlRef.current) URL.revokeObjectURL(prevObjectUrlRef.current);
  prevObjectUrlRef.current = newUrl;
  setModelUrl(newUrl);
  e.target.value = ''; // allow same file reload
};
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file system → Three.js GLTFLoader | User selects arbitrary .glb/.gltf file via OS file picker |
| Object URL → useGLTF cache | Blob URLs passed to useGLTF may share cached scene graphs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | GLBModel ErrorBoundary | mitigate | ErrorBoundary catches parse errors; setModelUrl(null) reverts to skeleton; no user-supplied code executes |
| T-02-05 | Denial of Service | Object URL leak | mitigate | prevObjectUrlRef tracks current object URL; URL.revokeObjectURL(prevUrl) called before each new load |
| T-02-06 | Tampering | Same-URL GLTF cache mutation | mitigate | After auto-fit, position/scale are applied to the group wrapper, not to scene internals — avoids shared scene graph mutation |
</threat_model>
